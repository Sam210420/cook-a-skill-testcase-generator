# AI Competition 2026
## Test Case Generator

## 1. Problem
Manual test case writing is time-consuming and often misses edge cases.

## 2. Solution
Build an AI-powered Test Case Generator that converts requirements into structured test cases automatically.
## 2.1 Input
- Requirement text
- User story
- API documentation
- UI description

## 2.2 Output
- Structured test cases table
- Positive cases
- Negative cases
- Edge cases
- Exportable CSV file

## 3. Core Features
- Generate functional test cases
- Generate negative test cases
- Generate edge cases
- Export to CSV/Excel
- Automation-ready format (Cypress / Playwright)

## 4. Target Users
- QA Engineers
- Testers
- Product Managers

## 5. Tech Stack
- Frontend: NextJS
- Backend: NodeJS
- AI: OpenAI / Claude API
- Hosting: Vercel

## 6. 5-Day Plan
Day 1: Spec + Setup
Day 2: Core generation logic
Day 3: Improve output quality
Day 4: Export feature + UI polish
Day 5: Demo & presentation
## 7. Competitive Advantage
- Focused on QA workflow
- Structured output instead of raw AI text
- Export-ready for real testing environments
